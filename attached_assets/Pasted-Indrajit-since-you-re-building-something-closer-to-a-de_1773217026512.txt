Indrajit — since you’re building something closer to a **decision intelligence platform** (not just dashboards), the UI should follow the style used by **Palantir Foundry, McKinsey Decision Intelligence, Salesforce Tableau Pulse, etc.**

Key principles for this style:

• **Decision-first layout** (not chart-first)
• **Hierarchy of insight → diagnosis → action → impact**
• **Minimal but powerful screens**
• **Operational workflows embedded in analytics**

Below is a **refined SaaS-style UI structure** tailored specifically for a **Copper Churn Intelligence Platform for a US ISP**.

# 8. Visual Design Principles (McKinsey / Palantir Style)

### Color Strategy

```
Primary: dark navy
Accent: copper orange
Neutral: grey
```

Risk colors

```
High risk → red
Medium → amber
Low → green
```

---

### Chart Rules

Avoid clutter.

Use:

```
line charts
bar charts
heatmaps
waterfall charts
cohort curves
```

Avoid:

```
pie charts (too many)
```
